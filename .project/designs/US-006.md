# US-006 — Papier & Crayon Foundations

## Objective

Create the first shared token set for the product so later UI stories can reuse a single palette, typography model, spacing scale, and responsive shell behavior.

## Tokens

- Palette: ivory canvas, white surface, soft beige muted surface, charcoal text, stone secondary text, anthracite accent, sand highlight.
- Typography: serif display for hero headings, sans body for interface copy, classic serif for generated document content, mono for labels and metadata.
- Spacing: compact `xs/sm`, readable `md/lg`, layout `xl/2xl/3xl/4xl`.

## Base Component Guidance

- `AppShell` should read like a paper sheet placed on a desk rather than a generic dashboard card.
- Supporting cards should stack vertically on mobile and move into columns only after the tablet breakpoint.
- Borders stay thin and quiet; shadows remain light and low-contrast.
