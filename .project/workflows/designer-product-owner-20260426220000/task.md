# Task — Full UX/Design Overhaul + Backlog & Sprint Update

## Source
Ad hoc — user feedback session 2026-04-26

## Request (verbatim intent)
The current app is a mess in terms of design. The mobile-first approach produces an awful result — nobody does CVs on mobile. We need a full redesign as a **desktop-first** web app with responsive fallback.

Key problems identified:
1. **Missing intermediate screens** — too much info packed onto single screens
2. **No table views** — candidates, interviews, past entries need sortable/filterable tables
3. **Interview mode is broken UX** — far from a natural call experience; user must press a button every time; the interview agent always gets appended text (no continuity in conversation)
4. **Puck Editor misplaced** — should be full-screen, admin-only; regular users only need to adjust data, not use Puck
5. **Too much text everywhere** — information density is too high; screens need breathing room
6. **Navigation needs rework** — must be clear, minimalist, understandable; inspired by shadcn clean aesthetic
7. **Backlog and sprints must be updated** — create new redesign-focused sprints starting from 016 (sprint 015 is the last sprint in the old numbering)

## Design inspiration
shadcn — minimalist, clean, desktop-first

## Scope
- Designer: full UX redesign spec covering screen architecture, navigation, interview mode, Puck placement
- Product Owner: update backlog (new epic + stories), create sprints 016+ focused on redesign delivery

## Constraints
- Must align with `.project/vision.md` or flag as explicit product decision
- WCAG 2.1 AA must be maintained
- Do not invent personas absent from vision
